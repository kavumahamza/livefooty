## `usePoll` API

```js
const { data, lastUpdatedAt, error, loading } = usePoll(url, intervalMs)
```

- `data` — parsed JSON from last successful fetch (null until first success)
- `lastUpdatedAt` — `Date.now()` timestamp of last successful fetch
- `error` — Error object if last fetch failed, else null
- `loading` — true until first response (success or failure)

**Backoff behaviour:** On any fetch failure, the hook sets `skipNextRef.current = true`. The next interval tick detects this and skips the fetch (resetting the flag), effectively doubling the wait after an error before retrying. This is intentionally minimal — one skipped tick, no exponential ramp.

**Cleanup:** Each effect registers an `AbortController`. On unmount (or url/intervalMs change), cleanup: `clearInterval`, `controller.abort()`, and `mountedRef.current = false`. Any in-flight fetch that resolves after abort is silently dropped (AbortError guard in catch).
